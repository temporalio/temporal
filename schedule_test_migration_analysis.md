## Analysis of the 6 DIFFs

None are substantive. All are explainable structural changes:

1. **TestSchedule_InternalTaskQueue (7->0) + subtests (0->3, 0->4)**: The old file uses `s.Run(...)` which the tool doesn't detect as a subtest boundary (it looks for `t.Run`). The new file uses `t.Run(...)`, so the tool correctly attributes assertions to the inner subtests. Total is preserved: 7 = 3 + 4.

2. **TestScheduleFunctionalSuite (2->0)**: These are two `s.NoError(err)` calls in the old suite runner for namespace registration. In the new code, `testcore.NewEnv` handles namespace creation internally and calls `t.Fatalf` on failure -- so these assertions are replaced by framework internals, not lost.

3. **getScheduleEntryFomVisibility (1->0) + getScheduleEntryFromVisibility (0->1)**: Typo fix in the function name. Same assertion, 1 = 1.
